Plume CSS

Welcome to Plume CSS, a lightweight and highly themeable CSS Micro-Framework. Plume is meant to be the lowest design layer on your app, helping you to grow solid and change easily. You should use Plume if you don't want to bring all the larger frameworks features to your application, or if you want to code everything by yourself and need a solid and highly customizable start point. Plume was inspired by projects such as Skeleton, Milligram, MiniCss and Material, encompassing some practices found in all of them. On this page you'll find an interactive Theme Editor, features overview, visual reference and a list of utilities and custom properties. Here are some of Plume's coolest characteristics:

Completly javascript free. Is just the pure and simple CSS. You can test it by disabling javascript and reloading this page. The theme editor wont be available, but Plume will be intact.
üîå   Plug and play
Load the lib, load your custom theme if you have one, and you're ready to go. Plume is already normalized. You can start to build your application with a minimal and solid design foundation.
üñå   Highly themeble
Builded with 70+ CSS custom properties (vars). Minimal and easy to extend and customize. Theme constructor with imediate visual feedback available (Button on the top left corner).
‚àà   Completely scoped
Styles are applied under the plume class, and custom properties are prefixed. You can add plume on an existent project with no harm and no side effects on the outer scope.
üíé   Decoupled themes
Custom themes are just a set of CSS Custom properties. You can load them dinamically or scope them to have multiple themes on the same application - even same page.
‚ü∑   Fluid typography
Typography is based on Material guidelines, and scales with the viewport width. The typography global definitions are pretty easy to tune.
üìê   No grid system
You can easily build any kind of layout with CSS Flexbox and/or Grid. If you still prefer to use a grid system lib, there are a plenty of them avaible out there, there is no need to reinvent the wheel.
üéØ   No bunch of classes
Despite a large set of utilities, Plume applies the style on bare elements. Classes and custom attributes are used to alter the default style and behavior.
üí°   Lib, framework or boilerplate
You can use plume as a lib, by adding it on your application as a base design layer. As a framework, by growing from Plume, or as a boilerplate, by cloning Plume's source code and starting from there.
‚ôø   Accessible
The outlines were preserved. The open architecture lets you freely to apply accessible best pratices and semantic HTML, and Plume has a ready made Accessible Custom Theme that you can toggle on your apps.
‚ú™   Unopinionated architecture
There is no html imposition, just pure and simple elements style. You decide your html architecture. You decide how to code your design. You have all the responsability, also all the freedom.

Getting Started

You can download Plume on the button below, then add the plume.min.css file on the dist folder to your project. Or you can add via NPM, then import Plume on your project. For examples and advanced load methods, please check the Documentation. 

npm install plume-css
